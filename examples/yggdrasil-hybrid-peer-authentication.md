## Overview

This example demonstrates how to use keys generated by `generate_joshua_kyber_key.py --hybrid` for **Yggdrasil** peer authentication and secure mesh communication.

Yggdrasil core uses Ed25519-based node identities. The hybrid keys generated here provide:

- **Classical X25519**: Immediate compatibility and high performance for today’s deployments.
- **Post-Quantum Kyber-1024**: Future-proofing against quantum attacks ("harvest now, decrypt later").

**Recommended approach in Nexus**:
1. Use the **X25519 public key** for Yggdrasil peer identification / whitelisting where supported.
2. Use the **full hybrid pair** (especially Kyber) for higher-layer authenticated encryption on top of Yggdrasil links (application-level or Noise protocol).

This gives defense-in-depth while the core mesh remains stable.

## Step 1: Generate Hybrid Keys

```bash

python3 generate_joshua_kyber_key.py --hybrid --output-dir ./nexus-keys
```

You will receive a file like:
`joshua_hybrid_x25519_kyber1024_<composite-fingerprint>.key`

**Extract the relevant sections**:
- `NEXUS X25519 PUBLIC KEY` (base64)
- `NEXUS X25519 PRIVATE KEY` (base64) — handle with extreme care
- `NEXUS KYBER1024 PUBLIC KEY`
- `NEXUS KYBER1024 PRIVATE KEY`

## Step 2: Yggdrasil Peer Identification (Classical Path)

Yggdrasil nodes identify each other via their public key (derived from the node’s private key).

While Yggdrasil’s primary identity is Ed25519, you can use the **X25519 public key** from the hybrid output for:

- Peer whitelisting / known-peers lists
- Custom application-level authentication
- Future NovaNet / QNET extensions that support X25519 or hybrid identities

### Example: Adding a known peer using the X25519 public key

In your `yggdrasil.conf` (or via `yggdrasilctl`):

```yaml
Peers:
  - uri: "tcp://203.0.113.50:12345"
    public_key: "<paste X25519 PUBLIC KEY base64 here>"   # From hybrid key file

# Or in AllowedPublicKeys style (if your Yggdrasil build supports it)
AllowedPublicKeys:
  - "<X25519 public key base64>"
```

**Note**: Standard Yggdrasil node identity remains Ed25519. Use the X25519 key as an additional authenticated identifier for Nexus-specific peers.

## Step 3: Higher-Layer Hybrid Authentication (Recommended for Full Security)

For true hybrid / post-quantum protection, perform an additional authenticated key exchange **over** the Yggdrasil link:

### Recommended Pattern

1. Establish basic Yggdrasil connectivity (encrypted at link layer).
2. On top of the Yggdrasil socket / stream, run a **hybrid Noise handshake** or custom protocol:
   - Use X25519 for the classical part of the handshake (IK or IKpsk2 pattern).
   - Use Kyber-1024 for the post-quantum key encapsulation.
3. Derive session keys from both.

This protects against both classical and quantum adversaries.

### Pseudocode Example (Python)

```python
# After connecting via Yggdrasil socket
sock = yggdrasil_connect(peer_address)

# Load your hybrid private keys
x25519_priv = load_x25519_private_key(...)      # from hybrid .key file
kyber_priv   = load_kyber_private_key(...)        # from oqs object or exported

# Perform hybrid handshake
classical_shared = x25519_dh(x25519_priv, peer_x25519_pub)
kyber_shared   = kyber_decapsulate(kyber_priv, peer_kyber_ct)

session_key = hkdf(classical_shared + kyber_shared)
# Now use session_key for authenticated encryption (e.g. ChaCha20-Poly1305)
```

**Libraries**: `cryptography` (X25519) + `oqs` (Kyber) make this straightforward.
